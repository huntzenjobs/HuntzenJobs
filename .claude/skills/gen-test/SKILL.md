# Generate Tests

Automatically generate comprehensive test suites for Python (pytest) and TypeScript (Vitest/React Testing Library).

## Usage

`/gen-test <file_path>`

Examples:
- `/gen-test backend/src/api/routes/jobs.py`
- `/gen-test frontend-next/src/components/jobs/search-form-inline.tsx`
- `/gen-test backend/src/agents/job_scout/main_agent.py`

Or generate tests for entire module:
- `/gen-test backend/src/api/routes/` - All route tests
- `/gen-test frontend-next/src/components/` - All component tests

## What this skill does

1. 📖 Analyze source code
2. 🧪 Generate test cases
3. 🎯 Cover edge cases
4. 💾 Save test files
5. ✅ Run tests to verify

## Backend Tests (Python + pytest)

### Test File Location

```
backend/src/api/routes/jobs.py
  → backend/tests/api/test_jobs.py

backend/src/agents/job_scout/main_agent.py
  → backend/tests/agents/test_job_scout_agent.py
```

### Template Structure

```python
"""
Tests for jobs API routes.
Generated by Claude Code /gen-test skill.
"""

import pytest
from fastapi.testclient import TestClient
from unittest.mock import Mock, patch, AsyncMock

from src.main import app
from src.models.schemas import JobSearchRequest

client = TestClient(app)


class TestJobsSearch:
    """Test suite for /api/jobs/search endpoint"""

    @pytest.fixture
    def mock_job_scout(self):
        """Mock JobScoutAgent"""
        with patch('src.api.routes.jobs.ScoutAgentDep') as mock:
            mock_agent = AsyncMock()
            mock_agent.run.return_value = {
                "success": True,
                "jobs": [
                    {
                        "id": "test-1",
                        "title": "Python Developer",
                        "company": "Acme Corp",
                        "location": "Paris",
                        "score": 0.85
                    }
                ],
                "metadata": {
                    "total_raw": 150,
                    "total_deduplicated": 50
                }
            }
            yield mock_agent

    def test_search_jobs_success(self, mock_job_scout):
        """Test successful job search"""
        response = client.post(
            "/api/jobs/search",
            json={
                "job_title": "Python Developer",
                "country_code": "fr",
                "city": "Paris",
                "max_results": 50
            }
        )

        assert response.status_code == 200
        data = response.json()
        assert data["success"] is True
        assert len(data["jobs"]) > 0
        assert data["jobs"][0]["title"] == "Python Developer"

    def test_search_jobs_missing_title(self):
        """Test search with missing job_title"""
        response = client.post(
            "/api/jobs/search",
            json={
                "country_code": "fr"
            }
        )

        assert response.status_code == 422  # Validation error

    def test_search_jobs_invalid_country(self):
        """Test search with invalid country code"""
        response = client.post(
            "/api/jobs/search",
            json={
                "job_title": "Developer",
                "country_code": "INVALID"
            }
        )

        assert response.status_code == 422

    def test_search_jobs_with_radius(self, mock_job_scout):
        """Test search with radius_km parameter"""
        response = client.post(
            "/api/jobs/search",
            json={
                "job_title": "Developer",
                "country_code": "fr",
                "city": "Paris",
                "radius_km": 25
            }
        )

        assert response.status_code == 200
        # Verify agent was called with radius_km
        mock_job_scout.run.assert_called_once()
        call_args = mock_job_scout.run.call_args[1]
        assert call_args["radius_km"] == 25

    def test_search_jobs_rate_limit(self):
        """Test rate limiting (50 requests/minute)"""
        # Make 51 requests quickly
        for i in range(51):
            response = client.post(
                "/api/jobs/search",
                json={
                    "job_title": f"Developer {i}",
                    "country_code": "fr"
                }
            )
            if i < 50:
                assert response.status_code == 200
            else:
                assert response.status_code == 429  # Too many requests

    @pytest.mark.parametrize("include_remote,expected_providers", [
        (True, ["adzuna", "jsearch", "remoteok"]),
        (False, ["adzuna", "jsearch"])
    ])
    def test_search_jobs_remote_filter(self, mock_job_scout, include_remote, expected_providers):
        """Test include_remote parameter filters providers"""
        response = client.post(
            "/api/jobs/search",
            json={
                "job_title": "Developer",
                "country_code": "fr",
                "include_remote": include_remote
            }
        )

        assert response.status_code == 200
        # Verify correct providers were used
        call_args = mock_job_scout.run.call_args[1]
        assert call_args["include_remote"] == include_remote
```

### Template Structure

```typescript
/**
 * Tests for SearchFormInline component
 * Generated by Claude Code /gen-test skill
 */

import { render, screen, fireEvent, waitFor } from '@testing-library/react'
import userEvent from '@testing-library/user-event'
import { vi, describe, it, expect, beforeEach } from 'vitest'
import { SearchFormInline } from './search-form-inline'
import { huntzenApi } from '@/lib/api/huntzen-client'

// Mock dependencies
vi.mock('@/lib/api/huntzen-client')
vi.mock('@/contexts/subscription-context', () => ({
  useSubscription: () => ({
    canUse: vi.fn(() => true),
    getRemaining: vi.fn(() => 10),
    isFreePlan: false
  })
}))

describe('SearchFormInline', () => {
  const mockOnSearch = vi.fn()

  beforeEach(() => {
    vi.clearAllMocks()

    // Mock API responses
    vi.mocked(huntzenApi.getCountries).mockResolvedValue([
      { code: 'fr', name: 'France' },
      { code: 'us', name: 'United States' }
    ])

    vi.mocked(huntzenApi.searchCities).mockResolvedValue([
      'Paris',
      'Marseille',
      'Lyon'
    ])
  })

  it('renders all form fields', () => {
    render(<SearchFormInline onSearch={mockOnSearch} />)

    expect(screen.getByPlaceholderText(/métier ou poste/i)).toBeInTheDocument()
    expect(screen.getByPlaceholderText(/pays/i)).toBeInTheDocument()
    expect(screen.getByPlaceholderText(/ville/i)).toBeInTheDocument()
    expect(screen.getByText(/rechercher/i)).toBeInTheDocument()
  })

  it('validates required fields before search', async () => {
    render(<SearchFormInline onSearch={mockOnSearch} />)

    const searchButton = screen.getByText(/rechercher/i)
    fireEvent.click(searchButton)

    await waitFor(() => {
      expect(screen.getByText(/le métier est requis/i)).toBeInTheDocument()
      expect(screen.getByText(/le pays est requis/i)).toBeInTheDocument()
    })

    expect(mockOnSearch).not.toHaveBeenCalled()
  })

  it('performs search with valid data', async () => {
    const user = userEvent.setup()
    render(<SearchFormInline onSearch={mockOnSearch} />)

    // Fill job title
    const jobInput = screen.getByPlaceholderText(/métier ou poste/i)
    await user.type(jobInput, 'Python Developer')

    // Select country
    const countryInput = screen.getByPlaceholderText(/pays/i)
    await user.type(countryInput, 'France')
    await user.click(screen.getByText('France'))

    // Click search
    const searchButton = screen.getByText(/rechercher/i)
    await user.click(searchButton)

    expect(mockOnSearch).toHaveBeenCalledWith({
      query: 'Python Developer',
      country: 'fr',
      location: '',
      includeRemote: true
    })
  })

  it('disables city autocomplete until country selected', async () => {
    render(<SearchFormInline onSearch={mockOnSearch} />)

    const cityInput = screen.getByPlaceholderText(/ville/i)
    expect(cityInput).toBeDisabled()

    // Select country first
    const countryInput = screen.getByPlaceholderText(/pays/i)
    await userEvent.type(countryInput, 'France')
    await userEvent.click(screen.getByText('France'))

    await waitFor(() => {
      expect(cityInput).not.toBeDisabled()
    })
  })

  it('shows radius slider only when city is selected', async () => {
    render(<SearchFormInline onSearch={mockOnSearch} />)

    // Initially no radius slider
    expect(screen.queryByText(/rayon/i)).not.toBeInTheDocument()

    // Select country and city
    const countryInput = screen.getByPlaceholderText(/pays/i)
    await userEvent.type(countryInput, 'France')
    await userEvent.click(screen.getByText('France'))

    const cityInput = screen.getByPlaceholderText(/ville/i)
    await userEvent.type(cityInput, 'Paris')
    await userEvent.click(screen.getByText('Paris'))

    // Now radius slider appears
    await waitFor(() => {
      expect(screen.getByText(/rayon/i)).toBeInTheDocument()
    })
  })

  it('includes remote jobs by default', async () => {
    render(<SearchFormInline onSearch={mockOnSearch} />)

    const remoteCheckbox = screen.getByLabelText(/inclure jobs remote/i)
    expect(remoteCheckbox).toBeChecked()

    // Perform search
    await userEvent.type(screen.getByPlaceholderText(/métier/i), 'Dev')
    await userEvent.type(screen.getByPlaceholderText(/pays/i), 'France')
    await userEvent.click(screen.getByText('France'))
    await userEvent.click(screen.getByText(/rechercher/i))

    expect(mockOnSearch).toHaveBeenCalledWith(
      expect.objectContaining({ includeRemote: true })
    )
  })

  it('respects freemium quota limits', async () => {
    // Mock freemium user with no remaining searches
    vi.mock('@/contexts/subscription-context', () => ({
      useSubscription: () => ({
        canUse: vi.fn(() => false),
        getRemaining: vi.fn(() => 0),
        isFreePlan: true
      })
    }))

    render(<SearchFormInline onSearch={mockOnSearch} />)

    // Fill and submit
    await userEvent.type(screen.getByPlaceholderText(/métier/i), 'Dev')
    await userEvent.type(screen.getByPlaceholderText(/pays/i), 'France')
    await userEvent.click(screen.getByText('France'))
    await userEvent.click(screen.getByText(/rechercher/i))

    // Should NOT call onSearch
    expect(mockOnSearch).not.toHaveBeenCalled()

    // Should show error toast (check via toast mock if available)
  })
})
```

## Test Generation Strategy

### 1. Analyze Source Code

```
Input: backend/src/api/routes/jobs.py

Detected:
- 4 endpoints (POST /search, GET /search, POST /analyze-query, GET /market-insights)
- Rate limiting (@limiter.limit)
- Authentication (CurrentUserDep)
- Pydantic schemas (JobSearchRequest, JobSearchResponse)
- Dependencies (ScoutAgentDep)
```

### 2. Generate Test Cases

For each endpoint:
- ✅ Success case (200)
- ✅ Validation errors (422)
- ✅ Auth errors (401)
- ✅ Rate limit (429)
- ✅ Mock dependencies
- ✅ Edge cases

### 3. Run Tests

```bash
# Backend
cd backend
pytest tests/ -v --cov=src

# Frontend
cd frontend-next
npm run test

# Coverage report
npm run test:coverage
```

## Best Practices

### Backend (pytest)

1. ✅ Use fixtures for common setups
2. ✅ Mock external dependencies (API calls, DB)
3. ✅ Use parametrize for variants
4. ✅ Test async functions with pytest-asyncio
5. ✅ Aim for 80%+ coverage

### Frontend (Vitest)

1. ✅ Use React Testing Library (not Enzyme)
2. ✅ Test user interactions, not implementation
3. ✅ Mock API calls (MSW preferred)
4. ✅ Test accessibility (screen readers)
5. ✅ Snapshot tests for UI components
