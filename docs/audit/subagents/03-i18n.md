# Audit -- Internationalisation
Date : 2026-03-18
Score : 28/100

## Resume executif

L'internationalisation du projet HuntZen presente des lacunes critiques. Si les pages dashboard (jobs, assistant, CV, salons, saved-jobs, documents, recruiter-contact) et les composants principaux (sidebar, pricing-modal, landing-header, footer) utilisent correctement `useTranslations`, **6 pages entieres** sont 100% hardcodees en francais sans aucun appel i18n : about, FAQ, blog, temoignages, privacy, terms. Par ailleurs, **44 cles** presentes en FR sont absentes du fichier EN, et les fichiers ES/PT sont encore plus incomplets (sections `sidebar.nav.referral`, `dashboard.assistant.welcomeTeam*`, `quickAction*`, etc. manquantes). De nombreux toasts et placeholders restent hardcodes dans le code source.

## Incoherences wording

### Tutoiement / Vouvoiement melange en FR

Le fichier FR melange systematiquement tutoiement et vouvoiement :

**Tutoiement (landing page, hero, features, progression, retention, coach, assistant):**
- `hero.h1`: "Ton CV merite d'etre vu"
- `features.jobs.title`: "Des milliers d'offres triees rien que pour toi"
- `dashboard.assistant.welcomeMessage`: "Tu viens de rejoindre ton equipe carriere"
- `progression.progressMessage`: "Tu avances bien"
- `retention.*`: Toutes les notifications en tutoiement

**Vouvoiement (dashboard, pricing, auth, modal, profil):**
- `tools.title`: "pour reussir votre recherche d'emploi"
- `dashboard.jobs.form.subtitle`: "Remplissez les champs ci-dessous"
- `pricingModal.title`: "Debloquez toutes les fonctionnalites"
- `auth.login.subtitle`: "Connectez-vous pour acceder a votre espace"
- `profile.cancelDialog.description`: "Vous conserverez l'acces..."

**Verdict :** Incoherence forte. La landing page tutoie, le dashboard vouvoie. Il faut choisir un registre unique.

## BLOQUANTS

1. **6 pages publiques 100% hardcodees FR** (about, FAQ, blog, temoignages, privacy, terms) -- tout utilisateur non-francophone voit du francais
2. **44 cles manquantes en EN** -- sections hero, features, footer, taglines pricing absentes
3. **~35 toasts hardcodes en francais** dans les composants -- messages d'erreur/succes non traduits
4. **EN `documents.downloadButton` = "Upload"** au lieu de "Download" -- erreur de sens critique pour l'UX
5. **EN salons : "Inline" au lieu de "Online", "Property" au lieu de "Event"** -- traduction automatique non revue
6. **PT `experienceSenior` = "Idoso"** -- mot offensant (vieillard) au lieu de Senior

## IMPORTANTS

1. **Tutoiement/vouvoiement melange** dans fr.json -- landing page tutoie, dashboard vouvoie
2. **~20 placeholders hardcodes** dans le CV builder (step-education, step-skills, step-personal-info, step-experiences, step-summary)
3. **~10 aria-labels hardcodes en francais** -- accessibilite non traduite
4. **ES/PT encore plus incomplets** que EN -- sections welcome assistant, quick actions, referral absentes
5. **Valeurs vides** dans `pricingModal.plans.premium.featureNames[7-8]` pour les 4 langues
6. **PT contient `&#x0D;`** (retour chariot HTML) dans `payment.cancel.tipLabel`
7. **ES contient `<g id="1">`** dans `savedJobs.comingSoonTitle`

## AMELIORATIONS

1. Unifier le registre FR : choisir tutoiement OU vouvoiement pour toute l'app
2. Externaliser les contenus SEO (about, blog) via un CMS ou au minimum dans les fichiers i18n
3. Creer des cles generiques reutilisables : `common.close`, `common.cancel`, `common.error`, `common.retry`
4. Revoir toutes les traductions EN de la section salons (clairement du machine translate non relu)
5. Revoir les traductions PT globalement (melange portugais du Portugal et du Bresil)
6. Les pages admin peuvent rester en FR (back-office interne), mais documenter ce choix
