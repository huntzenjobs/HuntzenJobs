## Décisions techniques prises

1. **STATUS_CONFIG sans label** : les labels de statuts sont résolus via `t("status.{key}")` — évite la duplication entre enum et i18n
2. **formatDate inline** : déplacée dans le composant pour accéder à `t` par closure
3. **DOMAIN_LABELS supprimé** : remplacé par `t("domains.{domain}")` avec fallback sur `domain` brut
4. **TypeScript cast** : `t(\`status.\${key}\` as Parameters<typeof t>[0])` pour satisfaire le typage strict de next-intl
