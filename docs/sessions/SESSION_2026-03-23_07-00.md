- Cookie unifie `huntzen_referral_code` pour referrals ET promos (pas de nouveau cookie)
- Detection type par format : `HZN-[A-Z0-9]{6}` = referral, sinon = promo
- Codes promo en DB (pas appel Stripe direct sur endpoint public) pour eviter brute force
- POST pour validation (pas GET) pour securite
- Un seul code a la fois (le dernier remplace)
- Onboarding step conditionnel (skip si code deja present)
