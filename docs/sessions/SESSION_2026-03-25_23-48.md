## Ce qui a ete fait

Correction des 7 problemes critiques identifies dans `docs/audit/AUDIT_QUOTA_PARRAINAGE_COMPLET.md` (RC-1 a RC-7).

### RC-1 : _apply_quota_bonus colonnes inexistantes
- Les colonnes `cv_analyses_remaining`, `coach_seconds_remaining`, `job_searches_remaining` n'existent pas dans `usage_quotas`
- Solution : RPC `apply_quota_bonus` qui decremente les compteurs `*_used` (GREATEST(0, used - bonus))

### RC-2 : _apply_free_days non atomique
- SELECT + UPDATE separees = race condition possible
- Solution : RPC `extend_subscription_days` avec UPDATE atomique `current_period_end + N days`

### RC-3 : Race condition tier rewards
- PostgREST `upsert(ignore_duplicates=True)` genere `ON CONFLICT (id) DO NOTHING` (PK only)
- Ne couvre PAS l'index fonctionnel JSONB `(referrer_id, tier_index)`
- Solution : RPC `insert_tier_reward` avec bare `ON CONFLICT DO NOTHING` (couvre TOUS les index uniques)
- Index unique fonctionnel ajoute : `idx_referral_rewards_tier_unique`

### RC-4 : Types frontend incomplets
- `cv_adapt` et `cover_letter` manquants dans `QuotasData` interface
- Solution : Ajout des champs optionnels dans `use-subscription-api.ts`

### RC-5 : FeatureType incomplet
- `cv_adapt`, `cover_letter`, `coach` manquants dans le type union `FeatureType`
- Solution : Ajout dans `use-freemium-limits.ts` + configs dans `usage-counter.tsx`

### RC-6 : Quota manquant sur /adapt/pdf
- Endpoint sans verification de quota
- Solution : Ajout `_check_and_increment_quota(user_id, "cv_adapt")` dans `cv_adapter.py`

### RC-7 : Increment premature des quotas
- `increment_assistant_messages` appele AVANT le traitement LLM = credit consomme meme si echec
- Solution : Deplace l'increment APRES le succes dans 5 endpoints de `assistant.py`

## Decisions techniques prises

1. **RPCs SECURITY DEFINER** pour toutes les operations atomiques (bypass RLS, execution cote serveur)
2. **Bare ON CONFLICT DO NOTHING** via RPC (impossible via PostgREST pour index fonctionnels JSONB)
3. **Decrement des used** plutot qu'increment des remaining (colonnes remaining n'existent pas)
4. **Pattern check-before / increment-after** : quota check au debut, increment apres succes
5. **Migration unique** regroupant les 3 RPCs + index pour deploiement atomique
